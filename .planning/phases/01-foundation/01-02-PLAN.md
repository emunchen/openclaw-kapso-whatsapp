       - default: return `fmt.Errorf("unknown transcription provider %q (valid: openai, groq, deepgram, local)", provider)`

2. Create `internal/transcribe/transcribe_test.go` with table-driven tests covering all behaviors above. Use `config.TranscribeConfig{}` directly. For the "local" provider test with non-existent binary, set BinaryPath to a path that won't exist (e.g., `/nonexistent/whisper-cli`).

1. Add `DownloadMedia(url string, maxBytes int64) ([]byte, error)` method to `kapso.Client` in `internal/kapso/client.go`:
   - Create GET request to `url`
   - Set `X-API-Key` header to `c.APIKey`
   - Execute via `c.HTTPClient.Do(req)`
   - Check `resp.StatusCode != http.StatusOK` — on error, read up to 512 bytes of body for error message
   - Use `io.LimitReader(resp.Body, maxBytes+1)` to read body
   - If `int64(len(data)) > maxBytes`, return error `fmt.Errorf("media response exceeds size limit (%d bytes)", maxBytes)`
   - Return data on success
   - Import `io` if not already imported (it IS already imported)

   - Verify successful download returns correct bytes
   - Verify exact-limit response succeeds (maxBytes == len(body))
   - Verify over-limit response (maxBytes+1 body) returns error
   - Verify X-API-Key header is present in request
   - Verify non-200 response returns error with status code

Note: `DownloadMedia` takes the URL directly (not a media ID). The caller will have already called `GetMediaURL` to get the URL. Pass `maxBytes` as a parameter, not from Client state — keeps Client stateless re: transcription config.

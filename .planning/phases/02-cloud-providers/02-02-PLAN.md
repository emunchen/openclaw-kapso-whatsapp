  <name>Task 1: Deepgram provider and retry wrapper</name>
  <files>internal/transcribe/deepgram.go, internal/transcribe/deepgram_test.go, internal/transcribe/retry.go, internal/transcribe/retry_test.go</files>
  <behavior>
    Deepgram:
    - Sends POST to https://api.deepgram.com/v1/listen with binary audio body (not multipart)
    - Sets Content-Type header to NormalizeMIME(mimeType)
    - Sets Authorization header to "Token " + APIKey (NOT "Bearer")
    - Includes query params: model, smart_format=true, and language (when non-empty)
    - Omits language query param when Language config is empty
    - Parses response from results.channels[0].alternatives[0].transcript
    - Returns error when channels array is empty
    - Returns error when alternatives array is empty
    - Returns *httpError on non-200 response

    Retry:
    - Retries on 429 (Too Many Requests) status
    - Retries on 5xx (500, 502, 503) status
    - Does NOT retry on 400 (Bad Request)
    - Does NOT retry on 401 (Unauthorized)
    - Returns success result when inner succeeds on first try
    - Returns success result when inner succeeds after retry (e.g., 429 then 200)
    - Returns "transcribe failed after 3 attempts: ..." when all attempts exhausted
    - Respects context cancellation (does not retry after ctx.Done)
    - Applies per-transcription timeout wrapping entire retry span
    - sleepFunc is injectable for testing (no real delays in tests)
  </behavior>
  <action>
    1. Create `internal/transcribe/deepgram.go`:
       - Define unexported `deepgramProvider` struct: `APIKey string`, `Model string`, `Language string`, `HTTPClient *http.Client`.
       - `client()` helper returning HTTPClient or http.DefaultClient.
       - `Transcribe(ctx, audio, mimeType)`:
         - `norm := NormalizeMIME(mimeType)`
         - Build query params with `url.Values{}`: set "model" to t.Model, "smart_format" to "true". If t.Language non-empty, set "language".
         - Create request: POST to `"https://api.deepgram.com/v1/listen?" + q.Encode()` with `bytes.NewReader(audio)` as body.
         - Set `Authorization: Token ` + t.APIKey (NOT Bearer).
         - Set `Content-Type` to `norm`.
         - Execute request. On non-200: return `&httpError{StatusCode, Body}`.
         - Parse response JSON into nested struct: `results.channels[].alternatives[].transcript`.
         - If `len(channels) == 0 || len(channels[0].alternatives) == 0`: return `fmt.Errorf("deepgram response missing channels/alternatives")`.
         - Return `channels[0].alternatives[0].transcript`.

    2. Create `internal/transcribe/deepgram_test.go`:
       - Table-driven tests with httptest.NewServer:
         - "success": returns valid response, verify transcript extracted correctly.
         - "auth header": verify Authorization is "Token test-key" (not Bearer).
         - "content-type": verify Content-Type header matches normalized MIME.
         - "query params": verify model, smart_format=true, language in URL query.
         - "no language": verify language param absent when Language is empty.
         - "empty channels": mock returns `{"results":{"channels":[]}}`, verify error.
         - "empty alternatives": mock returns channels with empty alternatives, verify error.
         - "non-200": mock returns 503, verify *httpError returned.
       - Mock server should verify request body is raw audio bytes (not multipart).

    3. Create `internal/transcribe/retry.go`:
       - `isRetryable(err error) bool`: use `errors.As` to check for `*httpError`. Retryable if StatusCode == 429 or StatusCode >= 500.
       - `retryTranscriber` struct: `inner Transcriber`, `attempts int`, `base time.Duration`, `factor float64`, `jitter float64`, `timeout time.Duration`, `sleepFunc func(time.Duration)`.
       - `newRetryTranscriber(inner Transcriber, timeout time.Duration) *retryTranscriber` — defaults: attempts=3, base=1s, factor=2.0, jitter=0.25, sleepFunc=time.Sleep.
       - `Transcribe(ctx, audio, mimeType)`:
         - If timeout > 0: wrap ctx with `context.WithTimeout(ctx, r.timeout)`, defer cancel. This wraps the ENTIRE retry span.
         - Loop up to r.attempts times. Call inner.Transcribe. On nil error return result.
         - If error is not retryable OR last attempt: break.
         - Check ctx.Err() before sleeping — if context cancelled, break immediately.
         - Calculate delay with jitter: `delay + random(0, delay * jitter)`. Sleep via sleepFunc.
         - Multiply delay by factor for next iteration.
         - On exhaustion: return `fmt.Errorf("transcribe failed after %d attempts: %w", r.attempts, lastErr)`.

    4. Create `internal/transcribe/retry_test.go`:
       - Use a mock Transcriber (not HTTP server) for retry-level tests — cleaner isolation:
         - Define `mockTranscriber` that returns a sequence of results/errors.
       - Test cases:
         - "success first try": inner returns ("text", nil). Verify result is "text", inner called once.
         - "retry on 429": inner returns httpError{429} then ("text", nil). Verify result is "text", inner called twice.
         - "retry on 5xx": inner returns httpError{503} then ("text", nil). Verify result is "text".
         - "no retry on 400": inner returns httpError{400}. Verify error after 1 attempt (not 3).
         - "no retry on 401": inner returns httpError{401}. Verify error after 1 attempt.
         - "exhausted": inner returns httpError{429} three times. Verify "transcribe failed after 3 attempts" error.
         - "context cancelled": use cancelled context. Verify returns quickly without retrying.
       - Use no-op sleepFunc and short base (1ms) in all tests for speed.
